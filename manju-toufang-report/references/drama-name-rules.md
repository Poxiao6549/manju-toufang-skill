# 剧名归一化规则（AI 语义识别）

**目标**：把 adex 返回的计划名（快手 `campaignName`）/项目名（巨量 `projectName`），归一化为**干净的剧集名**，并把指向同一部剧的多条记录合并聚合。

**为什么必须 AI 做**：剧名本身含中文标点（逗号、冒号、感叹号），前缀/日期/人名缩写位置不固定，纯正则会把剧名截断或把同剧拆成多部。由 LLM 语义理解识别最准。

---

## 一、命名样本示例（结构说明用）

**快手计划名：**
```
自投-示例剧名A-_0708_103529- dir_1
自投-示例剧名B，带逗号-2-0715-ops-001
自投-IAA-示例剧名C，带逗号-1-复刻动漫-0619001
自投-100200300-示例剧名D-1-[日期]-ops-002
自投-示例剧名E：带冒号-0711_134316-定投女- dir_45
自投-0705_104549_001示例剧名F：带冒号-1
自投-示例剧名G-0711_120554-通投- dir_31
```

**巨量项目名：**
```
自投-示例剧名H：带冒号-女频-15-0717-1
自投-示例剧名F：带冒号-0703-171203-2-默认卡点
自投-ops-示例剧名I！带感叹号-0717-测试-1
自制-示例剧名J：带冒号0618-151146-1
奇树-示例剧名K，带逗号-0620-002518-1
自投0602-200219-2-示例剧名L，带逗号
```

## 二、结构拆解（人写命名的构成）

一条名字大致由这些部分拼成，**顺序和有无都不固定**：

```
[投放来源前缀] - [账户号] - [IAA标记] - 【剧名】 - [批次/集数] - [日期时间] - [投放方式] - [人名缩写] - [序号]
```

- **投放来源前缀**：`自投`、`自制`、`奇树`、`代投` 等（可能带尾数字如 `自投0602`）
- **账户号**：6位以上纯数字（如 `100200300`）
- **IAA标记**：`IAA`（变现类型标记，不是剧名的一部分）
- **【剧名】**：← **这才是我们要的**，含中文标点，可能很长
- **批次/集数**：`-1`、`-2`、`-15`、`第二集`
- **日期时间**：`0716`、`_0708_103529`、`0703-171203`
- **投放方式**：`定投`、`通投`、`定投女`、`女频`、`测试`、`新测`、`老素材新测`、`复刻动漫`、`默认卡点`、`激励+上下滑`、`最大转化`
- **人名缩写**：`dir`、`ops` 等（导演/其他人，**本期忽略**）
- **序号**：`001`、`_1`、`-3`

---

## 三、归一化步骤（AI 执行）

对每条 `groupName`：

1. **剥离前缀**：去掉开头的 `自投/自制/奇树/代投`（含可能粘连的数字/日期，如 `自投0602`）
2. **剥离 IAA / 账户号**：`IAA`、纯数字账户号都不是剧名
3. **剥离尾部噪声**：日期、批次序号、投放方式标签、人名缩写
4. **提取剧名主体**：剩下的中文短语即剧名。**保留剧名内部的标点**（`示例剧名C，带逗号`、`示例剧名E：带冒号` 要完整保留逗号冒号）
5. **跨记录合并**：语义判断哪些记录是同一部剧
   - 完全同名 → 合并
   - `IAA-示例剧名C` 与 `示例剧名C` → **同一部剧**，合并
   - `示例剧名F：带冒号`（无论后缀 `-默认卡点`/`-第二集`/`-通投`）→ 全部合并
   - **快手和巨量的同名剧也要合并**（一部剧两个平台都投）

## 五、实操建议

- **批量处理**：一次报告可能有几十上百条名字。把去重后的 `groupName` 列表一次性在推理中归一化（建立 `原始名 → 剧名` 映射），再用映射对数据做聚合，避免逐条调用。
- **保守合并**：拿不准两条是否同剧时，**宁可分开**也不要错误合并（错误合并会让某剧消耗虚高，误导管理层）。可在报告注明"部分剧集因命名差异可能未完全合并"。
- **兜底**：完全无法识别剧名的（如纯数字/乱码名），归入「其他/未识别」，不要丢弃其消耗。
- **标注来源**：报告末尾固定注明"剧名为AI根据计划名识别，可能与实际剧集命名存在偏差"。
- **前缀清单**（去除用，持续补充）：`自投`、`自制`、`奇树`、`代投`、`IAA`
- **投放方式标签清单**（去除用，持续补充）：`定投`、`通投`、`定投女`、`女频`、`男频`、`测试`、`新测`、`老素材新测`、`复刻动漫`、`默认卡点`、`激励+上下滑`、`最大转化`、`上下滑`、`定投男`

> 遇到清单外的新前缀/新标签，归一化时按语义判断处理，并**回来补充本文件**（skill_manage patch）。
